# ZKM 混淆流程說明

此資料夾集中管理 Zelix KlassMaster（ZKM）相依檔案與腳本，請依下列步驟設定：

1. **安裝 ZKM**：依官方指引安裝後，記錄實際安裝根目錄，例如 `/opt/ZKM`。
2. **設定環境變數**：在終端機或 CI 中輸入 `export ZKM_HOME=/opt/ZKM`（或使用 `-Dzkm.home=/opt/ZKM` 覆寫）。
3. **補齊 Kotlin/Gson/Log4j/Jetty 依賴**：Spring 透過 Kotlin 偵測、Gson `TypeToken`、Log4j pattern 轉換器與 Jetty Servlet Handler，請將下列檔案複製至 `docs/zkm/lib/`（若版本有變，請同步更新 `zkm-config.zkm` 的 `classpath`）：

4. **準備腳本**：向授權擁有者取得 `.zkm` 腳本，並存放為 `docs/zkm/zkm-config.zkm`。檔案可參考官方模板，內容需指定輸入 JAR（`${project.build.directory}/demo-hotel-backend-0.0.1-SNAPSHOT.jar`）與輸出位置。
5. **啟用混淆**：預設不執行 ZKM；如需混淆，請執行 `mvn clean package -Dzkm.skip=false`，Maven 會在 `package` 段呼叫 `java -jar ${ZKM_HOME}/lib/ZKM.jar @docs/zkm/zkm-config.zkm`。
6. **輸出產物**：請在腳本內指定輸出目錄（建議 `target/zkm`），並於後續部署流程改用混淆後的 JAR。

> 提醒：ZKM 腳本會包含授權與密鑰，請勿提交至版本控制系統；如需不同環境設定，可維持一份加密或 CI 專用的變體並在部署前複製。
